text-analysis-and-datavis / folder index

A worked example of deterministic textual analysis + 2D semantic-embedding visualization of a literary corpus, paired with a draft composer that runs the same analyses on a student's prose in real time. Built around Italo Calvino's Six Memos for the Next Millennium. Start with summary.md; everything else is here for browsing.

Live page: a-project-on-calvino-interface-3kqu.vercel.app/memos — try it: paste a draft into the composer and watch the dashed bars and black dots appear.

Source repo: bok-learning-lab/a-project-on-calvino — the production multi-app monorepo. This example focuses on one page of that repo (the /memos page, which lives in apps/mw-project-002/).

Top-level

  • summary.md — what this project is, the three moves worth noticing (two analyses side by side; offline corpus + live student projection; LLM-mediated OCR pipeline as substance), how it was built, what you can translate it to
  • CLAUDE.md — project-level instructions loaded by Claude Code on session start
  • index.md / index.html — this file

inputs/

The cleaned literary corpus the page's analyses run on — the output of the upstream OCR pipeline. Five Calvino memos in cleaned markdown, page-range metadata, and two critic essays on the unwritten sixth memo.

  • inputs/memos/
    • 1-lightness.md — Lightness (~8,140 words, ~56 minutes oral)
    • 2-quickness.md — Quickness (~6,838 words)
    • 3-exactitude.md — Exactitude (~10,420 words — Calvino's longest)
    • 4-visibility.md — Visibility (~5,265 words — Calvino's shortest)
    • 5-multiplicity.md — Multiplicity (~7,531 words)
  • chapters.json — page-range metadata from the Mondadori scan
  • inputs/critics/
    • codrescu-on-consistency.md — Andrei Codrescu's Los Angeles Review of Books essay on Calvino's unwritten sixth memo
    • d0d1x-consistency-last-memo.md — D0D1X's Medium essay reading Consistency as the last memo

operations/

Three architectural documents + two production scripts copied verbatim. No Claude prompts (the page itself has no LLM call) — but the upstream OCR pipeline is LLM-mediated, and the embedding pipeline calls Gemini.

  • ocr-pipeline.md — the 10-step OCR pipeline that produced the cleaned markdown. Three OCR engines per page; per-token vote; Claude-vision judge on disagreements; Gemini sweep + human review. Same cheap-vote / expensive-judge posture as oral-exam-practice-bot's two-stage follow-up
  • deterministic-text-analysis.md — the seven core metrics computed live in the browser plus the sentence-length distribution histograms. Every formula documented, with the reasoning for each measurement and why hand-rolled SVG rather than a chart library
  • embedding-map-explained.md — the offline-once + live-projection architecture. Why Gemini, why UMAP not t-SNE, why k-NN weighted centroid for live projection
  • build-embeddings.py — copied verbatim from production (127 lines). The Python script that produces embeddings.json
  • embed-api-proxy.js — copied verbatim (51 lines). The Vercel serverless function that proxies live student-paragraph embedding to Gemini

outputs/

The illustrative material — a written architecture snapshot and two walkthroughs of what the page produces.

  • _source-snapshot.md — the Vite app architecture, the relevant slice of the multi-app repo layout, the deployment story. Includes the note that this is one page of a larger messy repo, and the page lives in a Vite app inside a mostly-Next.js monorepo
  • example-page-experience.md — what the student sees when they visit /memos: the header, the eight bar charts (with the actual Calvino numbers walked through), the embedding map, the composer, the three reader panels, the footer
  • example-draft-projection.md — a worked 353-word student draft of the unwritten Consistency memo, run through both analyses. Every chart's numbers walked through; the four paragraphs' embedding-map landings explained; what the worked example demonstrates pedagogically

To translate the pattern to another corpus: drop your cleaned author corpus into inputs/memos/, run operations/build-embeddings.py to produce embeddings.json, port the seven metrics in operations/deterministic-text-analysis.md verbatim (most are author-agnostic). If your corpus is a scanned PDF, run the OCR pipeline first. The hand-rolled SVG charts and the k-NN live-projection logic don't need per-corpus tuning.
