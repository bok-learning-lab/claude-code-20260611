paper-to-teaching-materials / folder index

A worked example of using Claude Code to build and run teaching material for an ethics-of-AI session, grounded in Grant, Behrends & Basl (2025) on decision-subject ethics. Start with summary.md; everything else is here for browsing. For a polished walkthrough suitable for sharing with colleagues, see outputs/case-study.html.

Top-level

  • summary.md — what this project is, the framework it applies, how we built it, what you can translate it to
  • CLAUDE.md — project-level instructions loaded by Claude Code on session start
  • index.md / index.html — this file

inputs/

The source paper, in two forms.

  • grant_behrends_basl.pdf — the source paper (open access, CC BY 4.0)
  • grant_behrends_basl.md — markdown digest for fast reading and citation lookup

operations/

The Deep Research prompt that produced the contextualizing essay, plus the skills that operate on the paper.

  • deep-research-prompt.md — the prompt for an LLM Deep Research tool (Claude Deep Research, ChatGPT Deep Research, Perplexity Deep Research). Produces a 2,500–3,500-word contextualizing essay on decision-subject ethics, the Grant-Behrends-Basl framework, and how it lands across adjacent fields
  • operations/skills/
    • teaching-case/ — generate a student-facing case + instructor notes, engineered to surface a specific move
      • SKILL.md
      • examples/ — case-medical-bail-5.2.md (a §5.2 case) and case-decoy-admissions.md (the construct-validity stress test, designed to pass)
    • discussion-plan/ — Socratic sequence around a case
      • SKILL.md
      • examples/ — discussion-plan-C1.md (a plan built around the C1 recidivism case)
    • objection-audit/ — steelman a student argument, then diagnose against the framework
      • SKILL.md
      • examples/ — audit-S1.md, audit-S2.md, audit-S3.md
    • quiz/ — short-answer comprehension check on the core argument
      • SKILL.md
      • examples/ — end-of-class-quiz.md

outputs/

Produced artifacts — the published landing page, the synthetic case & student-argument corpus, and the demo runs of the skills against that corpus.

  • case-study.html — the published landing page; one-page document with anchor navigation, suitable for sharing with a colleague who is not going to clone the folder
  • README.md — design rationale for the synthetic corpus
  • walkthrough.md — conversational steps that produced this project
  • outputs/cases/ — four cases: C1 recidivism, C2 resume screener, C3 interpretable alt (the construct-validity stress test), C4 juror model
  • outputs/student-arguments/ — three engineered-wrong arguments: S1, S2, S3
  • outputs/discussion-plans/ — discussion-plan-C1.md
  • outputs/objection-audits/ — audit-S1.md, audit-S2.md, audit-S3.md
  • end-of-class-quiz.md

To run the demo: open this folder in Claude Code, then /teaching-case for a §5.2 case → /discussion-plan around that case → /objection-audit on a student argument that picks up the missed distinction → /quiz for the comprehension check. The throughline to show the room: at every step Claude is augmenting the teacher's reasoning, never deciding — which is the paper's own thesis, demonstrated rather than asserted.
