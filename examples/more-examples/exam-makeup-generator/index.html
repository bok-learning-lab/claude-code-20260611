exam-makeup-generator / folder index

A worked example of a standalone Claude Code skill that generates candidate replacement problems for a make-up exam from an existing original, runs an instructor interview about per-slot topic scope, iterates on feedback, and assembles the chosen candidates into a finished make-up exam in the original's format. CS20-tested — the worked run is the real Round-1 + Round-2 + Assembly trace on the Harvard CS20 spring-2026 final exam. Start with summary.md; everything else is here for browsing.

This is the gallery's first standalone-skill example. Where the other examples ship deployed Vercel pages or CLI-style projects, this one ships only the skill (~400 lines) plus a worked run. There is no live URL.

Installation: drop the skill into a project's .claude/skills/ directory (or user-level ~/.claude/skills/).

mkdir -p .claude/skills/exam-makeup
cp operations/skill.md .claude/skills/exam-makeup/SKILL.md
cp operations/skill.json .claude/skills/exam-makeup/skill.json

Then invoke in any Claude Code session: Use exam-makeup to generate candidates from path/to/my-final.tex.

Top-level

  • summary.md — what this project is, the three moves worth noticing (file-as-state, the interview phase, security-vs-parity), how it was built, what you can translate it to. Installation instructions at the top
  • CLAUDE.md — project-level instructions loaded by Claude Code on session start
  • index.md / index.html — this file

inputs/

The worked run's input — the original exam the skill operated on.

  • final_s.tex — the CS20 spring-2026 final exam, LaTeX source (with solutions). 10 problems, the structural anchor for the run
  • final_s.pdf — same exam, rendered, for visual reference

operations/

The skill itself (the substance of the example) plus a reader's entry point.

  • three-mode-state-machine.md — read this first. The skill's shape in one document: three modes, one driving file, one interview moment. The reader's framework before the substance
  • skill.md — the skill itself. ~400 lines, deliberately single-document. Defines the three modes, the interview phase, the generation rules, the iteration semantics, the assembly sanity checks, the format-agnostic read strategies
  • skill.json — the one-line manifest (name, description, version, author)

outputs/

The worked run's output, plus a reader's guide.

  • worked-run-summary.md — a guide to the two real-run artifacts: what happened in which mode, what to look for in each file, what the numbers mean
  • final_s_makeup_candidates.md — the real run. 1,741 lines, 10 slots, 37 candidates across two rounds. Complete LaTeX solutions per candidate. Instructor's 10 Keep checkboxes pre-checked. This is the curation log + state-driving artifact
  • final_s_makeup.tex — the assembled exam. The chosen-per-slot candidates assembled into a complete make-up exam in the original's LaTeX format. Passed all sanity checks

After installing, invoking the skill is a multi-round flow: first call writes the candidates file (Generation mode); the instructor edits the file in their editor (Keep / Drop checkboxes; feedback blocks); re-invoking the skill auto-detects the next mode based on file state (Iteration if feedback is present; Assembly if all Keeps are set). The instructor never has to remember which mode they're in — they edit the file (or don't), re-invoke the skill, the skill figures out what to do.
