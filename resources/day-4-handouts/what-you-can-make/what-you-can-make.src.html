What You Can Make with Claude faculty projects, ordered from familiar text toward unfamiliar code

Each row names a kind of thing you can make: quick examples on the left, one unpacked on the right — situation, inputs, operations, outputs, and the interface that fits (ChatCoworkCode). Start where you have expertise; climb when you're ready.

1Structured Text & Documents Most familiar

Examples

  • Email to a chair proposing a new course
  • Grant Specific Aims page from a project description
  • Committee report from raw meeting notes
  • Recommendation letter from a CV and a prompt
  • Syllabus skeleton from a reading list

Unpacked

Meeting notes → a clean report for the dean

SituationMessy notes from a faculty meeting; a structured report due tomorrow.
Inputsmeeting-notes.md — your raw bullet points.
OperationsOne prompt: "turn these into a report with a summary, decisions made, and action items with owners and dates."
Outputsreport.md, or a polished .docx on request.
Best fitChatCoworkCode if repeated monthly

Unpacked

A glossary your students can search and copy from

SituationOne page holding every key term in your course, searchable, works offline.
Inputsglossary.md — terms and definitions.
Operations"Build a single-file HTML page with a search box and click-to-copy, styled for comfortable reading."
Outputsglossary.html — one file you can email or host anywhere.

Course site for GENED 1049 East Asian Cinema

SituationA site pairing a cinematography glossary with demos built on the films you actually teach.

Unpacked

Live literature search via the Semantic Scholar MCP

SituationReal papers with real citation counts and DOIs — not citations the model might invent.
InputsYour research question; a free API key in .mcp.json.
Operations"Find recent papers on X, get details, find related work, save." Claude chains search → get-paper → recommend.
OutputsA markdown literature summary with real DOIs.
Best fitCodeMCP servers run here

Unpacked

Calvino's Six Memos, by the numbers

SituationStudents see literary qualities as a measurable space — and plot their own prose against it.
InputsThe five memos as text; definitions for each measure.
OperationsDeterministic text analysis + a 2D embedding map + a live draft composer, charts drawn in d3.
OutputsA live page where pasted prose appears in every chart in real time.
Best fitCode
6Agentic Pipelines & Systems Most complex

Examples

  • Close-reading a whole corpus with parallel subagents
  • A reproducible coding pipeline for interview transcripts
  • An automated weekly research digest (scheduled)
  • A recommendation engine grounded in institutional data

Unpacked

Naming every writer cited across 538 Dylan songs

SituationA close reading at corpus scale — every author named in any lyric, with the line quoted — not a keyword grep.
InputsThe full corpus of songs (or transcripts, or a long text).
OperationsSpawn parallel close-reading subagents, each told to read like a scholar and refuse to grep; aggregate.
OutputsOne aggregated JSON plus a prose writeup pairing each name with its quoted line.
Best fitCodesubagents, scale

The arc of the week in one page: everything above is the same loop — inputs in a folder, an operation that transforms them, outputs you can hand to someone. What changes as you descend is only how much of the loop is code. Worked versions of nearly every row live in this repo's examples/more-examples/.
